# Cosine Similarity of Neighborhoods (All Pairs, Batch)
### Contents
  * [Specifications](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-batch#_specifications)

Supported Graph Characteristics
Weighted edges  
---  
Homogeneous vertex types  
Algorithm link: [Cosine Similarity of Neighborhoods (All Pairs, Batch)](https://github.com/tigergraph/gsql-graph-algorithms/tree/master/algorithms/Similarity/cosine/all_pairs)
This algorithm computes the same similarity scores as the [Cosine similarity of neighborhoods, single source algorithm](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-single-source).
Instead of selecting a single source vertex, however, it calculates similarity scores for all vertex pairs in the graph in parallel.
Since this is a memory-intensive operation, it is split into batches to reduce peak memory usage. The user can specify how many batches it is to be split into.
## [](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-batch#_specifications)Specifications
```
CREATE QUERY tg_cosine_nbor_ap_batch(STRING vertex_type, STRING edge_type,
  STRING edge_attribute, INT top_k, BOOL print_results = true,
  STRING file_path, STRING similarity_edge, INT num_of_batches = 1)
gsql![copy icon](https://docs.tigergraph.com/_/img/octicons-16.svg#view-clippy)Copied!

```

### [](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-batch#_parameters)Parameters
Name | Description  
---|---  
`v_type` | Vertex type to calculate similarity for  
`e_type` | Directed edge type to traverse  
`edge_attribute` | Name of the attribute on the edge type to use as the weight  
`top_k` | Number of top scores to report for each vertex  
`print_results` | If `true`, output JSON to standard output.  
`similarity_edge` | If provided, the similarity score will be saved to this edge.  
`file_path` | If not empty, write output to this file in CSV.  
`num_of_batches` | Number of batches to divide the query into  
### [](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-batch#_output)Output
The result of this algorithm is the top _k_ cosine similarity scores and their corresponding pair for each vertex. The score is only included if it is greater than 0.
The result can be output in JSON format, in CSV to a file, or saved as a similarity edge in the graph itself.
### [](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-batch#_time_complexity)Time complexity
This algorithm has a time complexity of O(E), where E is the number of edges.
## [](https://docs.tigergraph.com/graph-ml/3.10/similarity-algorithms/cosine-similarity-of-neighborhoods-batch#_example)Example
